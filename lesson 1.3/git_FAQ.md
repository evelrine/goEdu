# Шпаргалка по основным командам Git

## 1. Настройка
| Команда | Описание |
|---------|---------|
| `git config --global user.name "Имя"` | Установить имя пользователя |
| `git config --global user.email "email@example.com"` | Установить email |
| `git config --list` | Показать текущие настройки |

## 2. Инициализация и клонирование
| Команда | Описание |
|---------|---------|
| `git init` | Создать новый репозиторий в текущей папке |
| `git clone <url>` | Склонировать удалённый репозиторий |

## 3. Основная работа с изменениями
| Команда | Описание |
|---------|---------|
| `git status` | Показать состояние файлов (изменённые, добавленные и т.д.) |
| `git add <файл>` | Добавить файл в индекс (staging area) |
| `git add .` | Добавить все изменения в индексе |
| `git commit -m "Сообщение"` | Создать коммит с сообщением |
| `git commit -am "Сообщение"` | Добавить все отслеживаемые файлы и создать коммит |
| `git diff` | Показать различия между рабочим каталогом и индексом |
| `git diff --staged` | Показать различия между индексом и последним коммитом |
| `git restore <файл>` | Отменить изменения в рабочем каталоге (не добавленные) |
| `git restore --staged <файл>` | Убрать файл из индекса, но оставить изменения |

## 4. История и логи
| Команда | Описание |
|---------|---------|
| `git log` | Показать историю коммитов |
| `git log --oneline` | Краткая история коммитов (одна строка на коммит) |
| `git log --graph` | История с визуализацией ветвления |
| `git show <хеш>` | Показать изменения конкретного коммита |
| `git blame <файл>` | Показать, кто и когда изменял каждую строку |

## 5. Ветвление (Branches)
| Команда | Описание |
|---------|---------|
| `git branch` | Список локальных веток |
| `git branch -a` | Список всех веток (включая удалённые) |
| `git branch <имя>` | Создать новую ветку |
| `git checkout <ветка>` | Переключиться на другую ветку |
| `git switch <ветка>` | Переключиться на ветку (альтернатива checkout) |
| `git checkout -b <ветка>` | Создать и переключиться на новую ветку |
| `git merge <ветка>` | Слить текущую ветку с указанной |
| `git branch -d <ветка>` | Удалить локальную ветку (после слияния) |пш
| `git branch -D <ветка>` | Принудительно удалить ветку |
| `git push origin --delete <ветка>` | Удалить ветку на удалённом репозитории |

## 6. Работа с удалёнными репозиториями
| Команда | Описание |
|---------|---------|
| `git remote -v` | Показать подключённые удалённые репозитории |
| `git remote add origin <url>` | Добавить удалённый репозиторий с именем origin |
| `git push -u origin <ветка>` | Отправить ветку на удалённый репозиторий и установить связь |
| `git push` | Отправить изменения в связанную удалённую ветку |
| `git pull` | Забрать изменения с удалённого репозитория и слить с текущей веткой |
| `git fetch` | Забрать изменения без автоматического слияния |
| `git clone <url>` | Клонировать удалённый репозиторий |

## 7. Отмена изменений и исправление
| Команда | Описание |
|---------|---------|
| `git reset --soft HEAD~1` | Отменить последний коммит, оставив изменения в индексе |
| `git reset --mixed HEAD~1` | Отменить последний коммит и убрать изменения из индекса (по умолчанию) |
| `git reset --hard HEAD~1` | Полностью удалить последний коммит и все изменения |
| `git revert <хеш>` | Создать новый коммит, отменяющий изменения указанного коммита |
| `git stash` | Временно сохранить незакоммиченные изменения |
| `git stash pop` | Восстановить последние сохранённые изменения и удалить из stash |
| `git stash list` | Показать список сохранений в stash |

## 8. Полезные команды
| Команда | Описание |
|---------|---------|
| `git tag <имя>` | Создать тег для текущего коммита |
| `git tag -a <имя> -m "сообщение"` | Создать аннотированный тег |
| `git push --tags` | Отправить все теги на удалённый репозиторий |
| `git cherry-pick <хеш>` | Применить изменения конкретного коммита в текущую ветку |
| `git rebase <ветка>` | Перебазировать текущую ветку на другую |
| `git reflog` | Показать историю перемещений HEAD (для восстановления потерянных коммитов) |
